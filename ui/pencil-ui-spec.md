# Pencil UI 设计稿学习笔记

设计源文件：`D:\self\Academic-Administration-System\ui\pencil-new.pen`

## 1. 整体设计方向

这套设计稿是一个偏教务后台的浅色玻璃拟态管理界面，核心特征不是重阴影和高饱和，而是：

- 大面积浅蓝白渐变背景
- 半透明白色面板叠加 `background_blur`
- 统一的圆角、细描边、轻阴影
- 深色正文 + 蓝色品牌强调
- 左侧双层导航 + 右侧主内容区的固定后台结构

它更接近「轻盈、安静、信息密度中等」的 SaaS 仪表盘，而不是传统厚重的表单后台。

## 2. 画板结构

`.pen` 文件里目前是 5 个完整页面画板，不是先抽组件库再组装的方式：

- `Exam Dashboard`
- `Teacher List`
- `Score Management`
- `Class Config`
- `Monitor Config`

所有页面都沿用同一套壳：

- 左侧 76px 一级图标栏
- 中间 232px 二级导航栏
- 右侧 1040px 内容区
- 页面总宽 1440px，高约 980px
- 外层留白基本从 `x=28 / y=28` 开始

## 4. 背景与外层容器

页面背景几乎一致：

- 主背景为线性渐变：`#F5F9FF -> #EEF5FF`
- 叠加左上角径向高光：`#DDEBFF -> transparent`
- 整体很克制，亮度高，但不纯白

这意味着前端实现时不建议直接用纯色背景，而要保留：

- 1 层主渐变
- 1 层柔和径向光斑
- 足够大的留白

  - 背景 `#EAF3FFCC`

## 6. 内容区排版

- 表格卡片

## 7. 卡片语言

这套设计的卡片语言非常统一，是最值得前端复用的部分。

### 普通卡片

常见样式：

- 圆角 `20px`
- 内边距 `18` 或 `24`
- 填充 `#FFFFFFB8` 或 `#FFFFFFCC`
- 描边 `#FFFFFFD8` / `#FFFFFFE0`
- 外阴影偏淡蓝灰
- 同时启用 `background_blur`

### 强调卡片

例如进度卡：

- 圆角 `22px`
- 使用浅蓝渐变背景
- 描边更偏蓝：`#D9E8FF`
- 阴影比普通卡片更明显一点

### 内层字段块

例如输入区域、信息块：

- 圆角 `14px` 或 `16px`
- 半透明白底：`#FFFFFF96` 或 `#FFFFFF99`
- 描边 `#E9F0F8`
- 高度常见 `42px`、`74px`、`132px`

## 8. 表单与工具栏样式

工具栏 `studentToolbar` 的结构很典型：

- 高度 `78px`
- 圆角 `20px`
- 填充 `#FFFFFFB8`
- 描边 `#FFFFFFD8`
- 内边距 `18`
- 内部筛选器高度统一 `42px`

筛选器/搜索框特征：

- 圆角 `14px`
- 背景 `#FFFFFF96`
- 描边 `#E9F0F8`
- 左右内边距 `14px`
- 占位/普通文字：`14px / #667085`

主按钮特征：

- 高度 `42px`
- 圆角 `14px`
- 蓝色线性渐变：`#0F6CBD -> #2E86DE`
- 白字，常见 `14px / 700`
- 图标与文字间距 `8px`

## 10. 信息提示与状态表达

这套稿子状态表达比较克制，主要靠颜色浅层区分，不靠粗暴高饱和。

常见方式：

- 蓝色表示当前激活、数量、主要 CTA
- 绿色用于完成度或成功状态
- 橙色用于警告类信息
- 说明文字统一用 `#667085`
- 提示行一般是 `13px` 左右，带 `info` 图标

进度条也延续这个策略：

- 轨道浅蓝 `#DCEBFF`
- 进度段使用品牌蓝
- 右上角会搭配小状态 pill

## 11. 字体层级总结

当前设计稿里的字号层级比较稳定：

- 34px：页面主标题
- 22px：模块标题、卡片大标题、导航标题
- 15px：二级导航项
- 14px：工具栏字段、表格正文、按钮文字
- 13px：面包屑、描述、计数、小提示
- 12px：Tag、小段补充信息

字重规律：

- 700：页面标题、主按钮文字、标签
- 600：模块标题、选中导航、表格主字段
- 500：面包屑、普通导航
- 400：说明文案、输入提示

## 12. 前端还原建议

如果后续要把这份 Pencil 稿还原到前端，建议优先抽这几类基础组件：

- `AppShell`
- `PrimaryRail`
- `SecondaryNav`
- `GlassCard`
- `SoftField`
- `GradientButton`
- `InfoHint`
- `DataTable`
- `TagPill`

实现时建议保留以下关键视觉，不然会很快“变味”：

- 背景渐变 + 径向光斑
- 半透明白面板
- 蓝灰色细阴影
- 20px 左右大圆角
- 描边比阴影更重要
- 强调色只集中在品牌蓝，不要引入太多额外高饱和颜色

## 13. 当前结论

这份 `pencil-new.pen` 已经形成了一套比较统一的后台视觉语言，重点不是复杂交互，而是：

- 稳定的后台壳结构
- 统一的玻璃卡片体系
- 清晰的蓝色品牌表达
- 易于继续扩展到学生、教师、班级、考试等模块

后续如果要继续基于这份稿子开发，我建议直接把上面的变量、卡片、表格、工具栏样式抽成前端 token 和基础组件，而不是逐页硬写。
